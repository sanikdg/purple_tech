# Database Design

## Overview

The system uses **MongoDB** as its primary database. MongoDB was selected for its flexibility with event-driven data, schema evolution capability, and native support for JSON-like documents that map directly to the application's data structures.

## Collections

### users

Stores registered system users with role-based access.

```javascript
{
  _id: ObjectId,
  name: String,          // required, max 100
  email: String,         // required, unique, lowercase
  password: String,      // required, bcrypt hashed, min 6
  role: String,          // enum: "admin" | "operator" | "viewer"
  isActive: Boolean,     // default: true
  createdAt: Date,
  updatedAt: Date
}
```

---

### cameras

Stores CCTV camera configuration and status.

```javascript
{
  _id: ObjectId,
  name: String,          // required
  location: String,      // required
  streamUrl: String,     // required (RTSP/HTTP URL)
  resolution: {
    width: Number,       // default: 1920
    height: Number       // default: 1080
  },
  fps: Number,           // default: 30
  status: String,        // enum: "online" | "offline" | "error"
  isActive: Boolean,     // default: true
  zones: [ObjectId],     // ref: Zone
  createdAt: Date,
  updatedAt: Date
}
```

---

### zones

Defines spatial regions within a camera's field of view for tracking.

```javascript
{
  _id: ObjectId,
  zoneId: String,        // required, unique (e.g., "entrance-01")
  zoneName: String,      // required (e.g., "Main Entrance")
  zoneType: String,      // enum: "entrance" | "exit" | "section" | "checkout" | "restricted"
  polygonCoordinates: [  // min 3 points
    { x: Number, y: Number }
  ],
  description: String,
  cameraId: ObjectId,    // ref: Camera
  isActive: Boolean,     // default: true
  createdAt: Date,
  updatedAt: Date
}
```

**Example Zones**: Entrance, Exit, Lipstick Section, Skincare Section, Makeup Section, Checkout Counter, Restricted Area

---

### events

Stores all system events generated by the AI pipeline.

```javascript
{
  _id: ObjectId,
  eventType: String,     // enum: see EVENT_TYPES in eventTypes.js
  cameraId: ObjectId,    // ref: Camera, required
  trackId: ObjectId,     // ref: Track
  zoneId: ObjectId,      // ref: Zone
  timestamp: Date,       // default: Date.now, indexed
  payload: Mixed,        // flexible event-specific data
  confidence: Number,    // 0-1
  createdAt: Date,
  updatedAt: Date
}
```

**Indexes**: `{ eventType: 1, timestamp: -1 }`, `{ cameraId: 1, timestamp: -1 }`

---

### tracks

Stores customer tracking records through the store.

```javascript
{
  _id: ObjectId,
  trackId: String,       // required, unique (e.g., "TRK-00042")
  cameraId: ObjectId,    // ref: Camera, required
  zoneId: ObjectId,      // ref: Zone (current zone)
  startTime: Date,       // required
  lastSeen: Date,        // last frame timestamp
  duration: Number,      // seconds, computed
  currentPosition: {
    x: Number,
    y: Number
  },
  status: String,        // enum: "ACTIVE" | "LOST" | "EXITED"
  metadata: Mixed,       // flexible track-specific data
  createdAt: Date,
  updatedAt: Date
}
```

**Indexes**: `{ trackId: 1 }` (unique), `{ cameraId: 1, status: 1 }`, `{ startTime: -1 }`

**Relationships**:
- **Track → Events**: A track generates multiple events during its lifecycle (TRACK_CREATED, ZONE_VISITED, ZONE_LEFT, TRACK_LOST)
- **Track → Zones**: `zoneId` reflects the current zone; zone transitions generate events
- **Track → Cameras**: Every track originates from a specific camera
- **Track → Analytics**: Track data aggregates into daily analytics (visitor count, dwell time)

---

### analytics

Aggregated daily analytics per camera.

```javascript
{
  _id: ObjectId,
  date: Date,            // required, indexed
  cameraId: ObjectId,    // ref: Camera
  totalVisitors: Number, // default: 0
  peakHour: Number,      // 0-23
  averageDwellTime: Number, // seconds
  zoneBreakdown: [{
    zoneId: ObjectId,    // ref: Zone
    visitorCount: Number,
    avgDwellTime: Number
  }],
  hourlyDistribution: Map,  // hour → visitor count
  anomalies: [{
    type: String,
    description: String,
    detectedAt: Date,
    severity: String     // enum: "low" | "medium" | "high" | "critical"
  }],
  createdAt: Date,
  updatedAt: Date
}
```

**Indexes**: `{ date: -1, cameraId: 1 }`

---

### heatmaps

Stores individual position data points for heatmap generation.

```javascript
{
  _id: ObjectId,
  visitorId: String,     // required, indexed
  xCoordinate: Number,   // required (normalized 0-1 or pixel)
  yCoordinate: Number,   // required
  timestamp: Date,       // default: Date.now, indexed
  zoneId: ObjectId,      // ref: Zone
  cameraId: ObjectId,    // ref: Camera
  createdAt: Date,
  updatedAt: Date
}
```

**Indexes**: `{ timestamp: -1, zoneId: 1 }`, `{ cameraId: 1, timestamp: -1 }`
