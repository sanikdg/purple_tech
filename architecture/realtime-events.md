# Real-Time Events (WebSocket)

## Overview

The system uses **Socket.IO** for bidirectional real-time communication between the backend and frontend. All real-time events flow through the backend's Socket.IO server.

## Connection Flow

1. Frontend authenticates via REST API and receives a JWT token
2. Frontend establishes a Socket.IO connection to the backend
3. Client emits `join_dashboard` to join the dashboard room
4. Server pushes real-time events to all clients in the `dashboard` room

---

## Event Definitions

### `visitor_count_update`

| Property | Value |
|----------|-------|
| **Direction** | Server → Client |
| **Room** | `dashboard` |
| **Description** | Emitted when the active visitor count changes |

**Payload**:
```json
{
  "cameraId": "ObjectId",
  "currentCount": 15,
  "totalToday": 234,
  "timestamp": "2026-05-30T10:00:00.000Z"
}
```

---

### `new_event`

| Property | Value |
|----------|-------|
| **Direction** | Server → Client |
| **Room** | `dashboard` |
| **Description** | Emitted when any new event is generated by the AI pipeline |

**Payload**:
```json
{
  "eventType": "CUSTOMER_ENTERED",
  "cameraId": "ObjectId",
  "trackId": "ObjectId",
  "zoneId": "ObjectId",
  "timestamp": "2026-05-30T10:00:00.000Z",
  "confidence": 0.95,
  "payload": {}
}
```

---

## Client-Emitted Events

### `join_dashboard`

| Property | Value |
|----------|-------|
| **Direction** | Client → Server |
| **Description** | Client requests to join the dashboard room for real-time updates |

**Payload**: None

---

## Error Handling

- If the WebSocket connection drops, the client auto-reconnects (up to 5 attempts with 1s delay)
- On reconnect, the client re-emits `join_dashboard`
- The frontend displays a connection status indicator (Live / Offline)
